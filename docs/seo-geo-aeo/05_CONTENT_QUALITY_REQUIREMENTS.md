# 05 Content Quality Requirements

## Goal

Create content that is useful to users, differentiated from competitors, easy to read on mobile, and strong enough for SEO and AI-search extraction.

## 1. Every page needs a clear purpose

Before creating a page, define:
- primary audience
- search intent
- business objective
- conversion path
- target entity/topic
- primary query set
- supporting query set
- content type
- update cadence

## 2. Information gain requirement

Every important indexable page should include at least one of:
- original testing
- firsthand experience
- original data
- expert input
- proprietary methodology
- examples from real workflows
- unique comparisons
- current pricing/feature caveats
- decision-making criteria
- unique visuals/tables
- source-backed analysis

## 3. Avoid generic AI content

Do not publish content that:
- repeats common SERP answers without adding value
- has no author/reviewer accountability
- uses vague claims like “game-changing” or “revolutionary”
- has no examples
- has no current context
- lacks a clear opinion or decision guidance
- exists only because a keyword tool found a phrase

## 4. Recommended page structure

For review/comparison/guide pages:

1. H1
2. Updated/published date and author/reviewer where relevant
3. Short intro
4. Verdict / direct answer
5. Key takeaways
6. Good fit / not ideal for
7. Summary table
8. Main body sections
9. Comparison / alternatives
10. Pricing/limitations/compliance where relevant
11. FAQs
12. Sources / methodology

## 5. Summary box rules

Top summary boxes must be short and decision-oriented.

### Key Takeaways
- 3-5 bullets max
- no giant feature dump
- no repeated lower-page content
- no long paragraphs

### Good Fit / Not Ideal
- 3-5 bullets per side
- short phrases
- decision-oriented

## 6. Mobile readability

Use:
- short paragraphs
- clear H2/H3s
- bullets only where useful
- tables that work on mobile
- summary boxes
- comparison cards
- meaningful subheads

Avoid:
- dense walls of text
- excessive bullets
- overlong intros
- massive tables with no mobile treatment

## 7. E-E-A-T style quality

For important pages, include:
- who wrote/reviewed it
- why they are qualified
- what was tested or reviewed
- when it was last updated
- sources used
- limitations/caveats
- clear correction/update policy where relevant

## 8. AI-search readiness

Each major section should contain:
- a clear answer
- supporting explanation
- examples
- precise terminology
- entity names written consistently

Avoid making AI systems infer the main point from vague narrative.

## 9. Refresh rules

Refresh pages when:
- pricing changes
- product features change
- legal/compliance context changes
- SERP intent changes
- rankings decline materially
- competitors add better information
- AI answer engines begin citing weaker/wrong sources
- content becomes stale or unsupported

## 10. Deletion/consolidation rules

Consolidate or remove pages that:
- have no unique intent
- duplicate another page
- receive no impressions/traffic/links/conversions
- are thin and not strategically necessary
- cannibalize stronger pages
- cannot be improved into a useful asset

Do not delete pages with backlinks, conversions, or strategic value without redirect/noindex/canonical planning.
